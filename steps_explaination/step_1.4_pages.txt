Why We Did It (Theory):
-----------------------
A Page Map and UI Wireframe outline:
- Establishes URL Routing structures: We map our physical file layouts (e.g. app/views/home/index.php, app/views/service/detail.php) to friendly virtual URL paths.
- Optimizes UI/UX consistency: Separating layout blocks (header.php, navbar.php, footer.php, admin_header.php, admin_sidebar.php) allows us to reuse code. If we change a navigation link, we edit it once in navbar.php and it updates on every page.
- Directs Controller actions: It maps which controllers (e.g. ServiceController, AuthController, DashboardController) need which action views (e.g. index, create, edit, delete).
