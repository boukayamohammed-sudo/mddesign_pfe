Why We Did It (Theory):
-----------------------
The MVC pattern is a fundamental design pattern used in most modern web frameworks (Laravel, Symfony, CodeIgniter, Django, Ruby on Rails). Understanding it is essential for your PFE:
- Separation of Concerns: Each layer has one job. Models handle data, Views handle display, Controllers handle logic. This makes the code easier to debug and maintain.
- Front Controller Pattern: By routing ALL requests through a single index.php file, we prevent direct access to internal PHP files. A user cannot type 'yoursite.com/app/models/Admin.php' and see your database code.
- Reusability: The base Controller class provides a 'view()' method that every controller inherits. The base Model class provides database access that every model inherits. This avoids code duplication.
- Testability: When logic is separated, you can test Models independently from Views, and Controllers independently from database queries.
